# Чек-лист тестирования: Фильтры каталога товаров DNS

## 1. Подготовительный этап
Определить тестовый раздел: Выбрать категорию с большим количеством товаров (например, "Ноутбуки", "Смартфоны", "Комплектующие ПК") для полноты проверки.

Подготовить тестовые данные: Составить список товаров с известными характеристиками для проверки граничных значений фильтров.

Очистить кэш/куки: Перед началом тестирования очистить браузер или использовать режим инкогнито для исключения влияния предыдущих сессий.

Зафиксировать базовое состояние: Проверить, что каталог отображается без применения фильтров (все товары категории).

## 2. Отображение и доступность фильтров
### 2.1. Общее отображение
- [ ] Присутствие блока фильтров: Блок фильтров отображается на странице каталога (слева, сверху или в выезжающей панели на мобильной версии).
- [ ] Видимость всех фильтров: Отображаются все предусмотренные группы фильтров для выбранной категории (цена, производитель, характеристики, наличие, цвет и т.д.).
- [ ] Адаптивность: На мобильных устройствах фильтры доступны по кнопке "Фильтр" (или аналогичной) и корректно открываются/закрываются.
- [ ] Счетчики количества: Проверить, что рядом с названием фильтра или его значением отображается количество товаров, соответствующих данному критерию (если предусмотрено дизайном).

### 2.2. Динамическая подгрузка фильтров
- [ ] Актуальность списка: При выборе одной категории (например, "Ноутбуки") фильтры соответствуют характеристикам этой категории (появляются "Диагональ экрана", "Процессор"; пропадают нерелевантные, например, "Тип масла").
- [ ] Вложенные категории: При переходе в подкатегорию (например, "Игровые ноутбуки") набор фильтров должен обновляться в соответствии с товарами этой подкатегории.

## 3. Функциональное тестирование фильтров
### 3.1. Фильтр по цене
- [ ] Поля ввода: Поля "от" и "до" принимают только числовые значения.
- [ ] Ползунок (слайдер): Работает плавно, оба бегунка двигаются независимо, не пересекаются (минимальное значение не может быть больше максимального).
- [ ] Синхронизация: При перемещении ползунка значения в полях ввода обновляются, и наоборот.
- [ ] Корректный ввод: Ввод "1000" в поле "от", "20000" в поле "до" → применяется фильтр, показываются товары в этом диапазоне.
- [ ] Граничные значения: Ввод цены, равной минимальной/максимальной цене товаров в каталоге → товары отображаются корректно.
- [ ] Инверсия: Ввод в поле "от" значения большего, чем в поле "до" → ошибка валидации, фильтр не применяется.
- [ ] Пустые значения: Одно из полей пустое → фильтр работает по открытому диапазону (только "от" или только "до").
- [ ] Некорректный ввод: Ввод букв, спецсимволов, отрицательных чисел → поле не принимает значения или показывает ошибку.

### 3.2. Фильтр по производителю/бренду
- [ ] Одиночный выбор: Выбор одного бренда → в результатах только товары выбранного бренда.
- [ ] Множественный выбор: Выбор нескольких брендов → в результатах товары всех выбранных брендов (логика "ИЛИ").
- [ ] Чекбоксы: Состояние "выбран/не выбран" отображается корректно.
- [ ] Счетчики: Рядом с брендом отображается количество доступных товаров этого бренда (с учетом других активных фильтров).

### 3.3. Фильтр по характеристикам
Типы характеристик:

- [ ] Чекбоксы: Выбор одного или нескольких значений (например, "Диагональ: 15.6", "17.3").
- [ ] Радиокнопки: Выбор только одного значения из группы (если предусмотрено).
- [ ] Выпадающий список: Раскрывается, значения выбираются.
- [ ] Ссылки: Клик по значению применяет фильтр.

- [ ] Логика комбинирования: При выборе нескольких характеристик внутри одной группы (например, "Оперативная память: 8 ГБ" и "16 ГБ") → показываются товары, соответствующие любому из выбранных значений (логика "ИЛИ").
- [ ] Логика между группами: При выборе характеристик из разных групп (например, "Процессор: Intel" И "Оперативная память: 16 ГБ") → показываются товары, соответствующие ВСЕМ выбранным группам (логика "И").

### 3.4. Фильтр по наличию
Варианты наличия: Проверить работу фильтров:

- [ ] "В наличии" → только товары, доступные для покупки сейчас.
- [ ] "Под заказ" → товары, которые нужно заказать (отсутствуют в магазинах).
- [ ] "Ожидается" → предзаказ (если применимо).

- [ ] Комбинирование: Сочетание фильтра "В наличии" с другими фильтрами → результаты соответствуют обоим условиям.

### 3.5. Фильтр по цвету
- [ ] Отображение: Цвета отображаются в виде цветных квадратов, текста или иконок.
- [ ] Выбор: При клике на цвет товары соответствующего цвета отображаются корректно.
- [ ] Множественный выбор: Возможность выбрать несколько цветов (если поддерживается).

## 4. Комбинирование фильтров
- [ ] Последовательное применение: Применить фильтр A → применить фильтр B → результаты соответствуют A И B.
- [ ] Одновременное применение: Выбрать несколько фильтров из разных групп → результаты соответствуют всем условиям.
- [ ] Конфликтующие фильтры: Применить фильтры, которые не могут быть выполнены одновременно (например, "Бренд: Apple" и "Диагональ: 17.3 дюйма" для ноутбуков, если таких нет) → результат "Товаров не найдено" с корректным сообщением.
- [ ] Сброс отдельных фильтров: Снять один фильтр → результаты обновляются, остальные фильтры остаются активными.
- [ ] Сброс всех фильтров: Кнопка "Сбросить все" или "Очистить" → все фильтры сбрасываются, отображаются все товары категории.

## 5. Сортировка и фильтры
- [ ] Совместная работа: Применены фильтры → изменить сортировку (по цене, популярности, новизне) → результаты остаются отфильтрованными, но сортируются по новому правилу.
- [ ] Сохранение сортировки: При добавлении/удалении фильтров выбранная сортировка не сбрасывается (или сбрасывается согласно логике приложения).

## 6. Пагинация и подгрузка
- [ ] Обновление количества страниц: При применении фильтров общее количество страниц пагинации пересчитывается корректно.
- [ ] Позиция на странице: При применении фильтра, если текущая страница становится пустой (например, был на 3-й странице, а после фильтрации осталось 2 страницы), происходит переход на последнюю доступную страницу.
- [ ] Бесконечная подгрузка (если есть): При скролле подгружаются только товары, соответствующие активным фильтрам.

## 7. URL и состояние страницы
- [ ] Синхронизация URL: При применении фильтров параметры фильтрации отображаются в URL (например, `?price_min=1000&price_max=20000&brand=apple`).
- [ ] Прямой переход: Скопировать URL с параметрами фильтров, открыть в новой вкладке → фильтры применены, результаты соответствуют.
- [ ] Кнопка "Назад" браузера: Применить фильтры, нажать кнопку "Назад" → состояние страницы возвращается к предыдущему набору фильтров.
- [ ] Кнопка "Вперед" браузера: После возврата нажать "Вперед" → возвращаются ранее примененные фильтры.

## 8. Производительность
- [ ] Скорость применения: Применение фильтров (особенно множественных) происходит без длительных задержек (не более 2–3 секунд).
- [ ] Индикация загрузки: Во время применения фильтров отображается индикатор загрузки (спиннер, скелетон), чтобы пользователь понимал, что процесс идет.
- [ ] Количество запросов: При последовательном выборе фильтров не происходит избыточного количества запросов (оптимизировано через debounce/throttle).

## 9. Мобильная версия
- [ ] Открытие фильтров: Нажатие на кнопку "Фильтр" открывает модальное окно/шторку с фильтрами.
- [ ] Применение на мобильных: После выбора фильтров нажатие кнопки "Применить" закрывает шторку и обновляет результаты.
- [ ] Сброс на мобильных: Доступна кнопка "Сбросить все" внутри модального окна.
- [ ] Сенсорное управление: Ползунок цены удобен для пальца, чекбоксы имеют достаточный размер для нажатия.

## 10. Негативные сценарии
- [ ] Нет товаров по фильтрам: Отображается сообщение "Товары не найдены" или "Ничего не найдено". Предлагаются альтернативы (сбросить фильтры, перейти в другую категорию).
- [ ] Обрыв соединения: Применить фильтр при отключенном интернете → отображается сообщение об ошибке, страница не "падает", предыдущие результаты сохраняются.
- [ ] Медленное соединение: При медленном интернете фильтры не "зависают", есть индикация загрузки.
- [ ] Очень много товаров: Применить фильтр, возвращающий большое количество товаров (например, только ценовой диапазон) → пагинация/подгрузка работает корректно, страница не тормозит.

## 11. Особые сценарии для DNS
- [ ] Фильтр по наличию в магазине: Если выбран конкретный город, фильтр "В наличии" должен показывать товары, доступные в магазинах этого города, а не на складе в целом.
- [ ] Фильтр по рейтингу: Проверить работу фильтра по звездам (например, 4 звезды и выше) → результаты соответствуют.
- [ ] Фильтр по акции: Фильтр "Товары со скидкой" или "Акция" → показываются только товары, участвующие в акциях.
- [ ] Фильтр по новинкам: Показываются только товары с меткой "Новинка".

## 12. Итоговые End-to-End сценарии
Сценарий "Полный путь с фильтрацией":

1. Открыть категорию "Смартфоны".
2. Установить ценовой диапазон от 20 000 до 50 000 ₽.
3. Выбрать бренды: Samsung, Xiaomi.
4. Выбрать характеристику: "Объем памяти: 128 ГБ".
5. Применить фильтр "В наличии".
6. Ожидаемый результат: Отображаются смартфоны Samsung и Xiaomi, с памятью 128 ГБ, в ценовом диапазоне 20 000–50 000 ₽, доступные для покупки.
7. Отсортировать по цене (сначала дорогие).
8. Перейти на 2-ю страницу результатов.
9. Сбросить все фильтры.
10. Ожидаемый результат: Отображаются все смартфоны категории, пагинация восстановлена, сортировка сброшена (или сохранена согласно логике).

Сценарий "Комбинирование с поиском":

1. Ввести поисковый запрос "iPhone".
2. Применить фильтр по цене.
3. Ожидаемый результат: Результаты поиска отфильтрованы по цене.

Сценарий "Кросс-категорийная фильтрация":

1. Перейти в раздел "Ноутбуки".
2. Применить фильтр "Процессор: Intel Core i7".
3. Перейти в подкатегорию "Игровые ноутбуки".
4. Ожидаемый результат: Фильтр по процессору сохраняется, отображаются только игровые ноутбуки с процессором Intel Core i7.

## 14. Доступность (Accessibility)
- [ ] Клавиатурная навигация: Фильтры доступны и управляемы с клавиатуры (Tab, пробел, Enter).
- [ ] Скринридеры: Элементы фильтров имеют корректные aria-метки, состояние "выбран/не выбран" озвучивается.
- [ ] Контрастность: Текст фильтров и элементы управления имеют достаточную контрастность для слабовидящих.
